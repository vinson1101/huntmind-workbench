# 转化话术规则（Conversion Guidelines）

本文件定义招聘决策 skill 中与候选人触达和转化相关的输出要求。

## 必须输出的转化字段

你必须输出：

1. `hook_message`
   - 需要让候选人愿意回复
   - 不能只是普通问候
   - 要体现这个岗位对他的吸引点

2. `verification_question`
   - 用于快速识别真假能力
   - 必须具体、尖锐、可用于首轮判断

3. `message_template`
   - 可直接复制发送的完整联系话术
   - 要自然，不要过长
   - 需要体现岗位吸引力和针对性

4. `deep_questions`
   - 至少 3 个深问问题
   - 用于判断是否值得推进下一轮

## 设计原则

- 不是“分析性输出”，而是“可推进招聘动作的输出”
- 联系话术必须体现针对性，而不是通用模板
- 验证问题必须帮助快速识别真实能力，不要空泛
- 深问问题必须能帮助判断是否继续推进

## 不允许的做法

- 只输出普通问候，不体现岗位吸引点
- 只输出泛泛问题，无法区分真假能力
- 只输出过长模板，不可直接发送
- 只输出表面问题，没有下一轮验证价值

## 推荐方向

- 先抓候选人与岗位真正贴合的点做开场
- 第一问要能快速判断职责边界和真实 ownership
- 深问问题要覆盖：
  - 代表性项目
  - 真实负责部分
  - 业务理解
  - 推动落地能力
